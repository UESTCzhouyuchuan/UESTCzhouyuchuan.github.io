                E2E自动化测试Cypress入门

                玉川 2020-03-21 E2E Cypress

                # 前言 --我对自动化测试的理解

                我认为自动化测试是从人脑测试到程序测试的一种革命。通过模拟用户操作，判读啊是否达成预期结果，不管中间发生什么，这是软件工程非常典型的黑盒测试。

                cypress 非常强大，而且对开发者友好，提供很完善的 api，还提供了类似 promise 对象的链式调用，它因该是借鉴了 promise 实现的思想，在函数调用结束也返回了一种相同类似但状态发生改变的对象。我还很喜欢它的 get 函数，前端选手很轻易就能记着它并且拿来用，实现原理应该是通过 document.querySelector 获得 dom，然后通过该 dom 初始化一个对象，然后返回出来，模拟操作在该对象上进行，然后反映到 DOM 和视图中。

                # 安装使用

                由于国外源太慢，使用公司 VPN 也无济于事，为 npm 或者 yarn 指定 cypress 的下载路径。

                # 使用 npm

                如下，在～/.npmrc(使用 npm)文件中添加

                CYPRESS_INSTALL_BINARY=https://npm.taobao.org/mirrors/cypress/4.1.0/mac/cypress.zip
                1

                新建一个 test_runner 目录，先 npm init，然后下载

                npm install -D cypress
                1

                # 使用 yarn

                如果你使用 yarn，在～/.yarnrc 文件中

                CYPRESS_INSTALL_BINARY "https://npm.taobao.org/mirrors/cypress/4.1.0/mac/cypress.zip"

                yarn 也是同样道理

                yarn add -D

                注意，要根据版本和设备类型下载，去@https://npm.taobao.org/mirrors/cypress查看最新版

                # cy 的常用的 API

                API 地址https://docs.cypress.io/api/commands/and.html#Syntax

                1. get 函数，参数是 css 选择器，后续操作都是在他返回的对象上进行的。get 可以获得多个结果，使用 each 函数对每个 ele 进行操作，如果直接对 get 的结果进行动作，默认选择第一个。
                2. type 函数，模拟人向输入框中输入数据，参数是要输入的数据。type 也可以模拟键盘输入，格式为{键盘动作}，
                3. click()，模拟人点击。
                4. should，先行断言，判断符合预期，根据第一个参数决定接下来的参数，第一个参数代表状态，如下
                5. be 系列be.impty be.hidden be.visible be.check be.select
                6. has 系列has.class has.value has.id has.attr has.focus
                7. 函数还可以是一个函数，会向函数注入当前对象
                8. contians()，获得满足 contians 内容的第一个 DOM。包含内容的判断依据是 text 值或者 value 值。content 可以是 string，number，或者强大的正则表达式。官方文档https://docs.cypress.io/api/commands/contains.html#Syntax。注意：contains 判断的元素必须是其前者的字元素或者本身。

                # Cy 实践终结

                首先，cypress 的每一步操作都是异步进行的，它封装的对象类 promise 对象，比 promise 多了一些成员方法属性。

                • it 作为一组动作，会在上一个动作完成后加入执行队列。
                • beforEach 函数接受一个执行函数，回注入到每一个 it 之前。
                • 如果需要登陆，必须在每一个 case（即 it）之前通过 beforeEach 注入通过 cy.setCookie 设置 cookie，完成登陆。或者完成登陆请求保存 cookie。
